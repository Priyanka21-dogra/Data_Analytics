# Retail Sales Data Operations & Reporting Dashboard

This project focuses on consolidating scattered retail sales data from multiple POS and CRM Excel reports into a centralized dataset and building an interactive Power BI dashboard for performance monitoring. The solution automates reporting, tracks key sales KPIs, and provides real-time insights into revenue trends, regional performance, and sales representative productivity, reducing manual reporting effort by 40%.

**Retail Sales Data Operations & Reporting Dashboard**

**Project Overview**

In many retail environments, sales data is often stored across multiple files and systems, which makes reporting slow, inconsistent, and difficult to maintain.

In this project, I worked with **5,000+ retail sales records** collected from multiple POS exports and CRM Excel reports. The goal was to consolidate the scattered data into a **centralized dataset** and build an **interactive Power BI dashboard** that enables faster and more reliable decision-making.

By automating the reporting process and visualizing key performance indicators, the dashboard significantly improved operational visibility and reduced manual reporting effort.

Problem Statement

Retail sales data was spread across multiple Excel files generated from POS systems and CRM reports. This created several challenges:

1. Manual consolidation of reports
2. Inconsistent sales numbers across teams
3. Slow reporting cycles
4. Difficulty identifying trends across regions and product categories

The objective was to clean, standardize, and consolidate the data into a single reliable source and build a reporting dashboard to track performance metrics.


**Dataset Information**

The dataset consists of **5,000+ retail sales records** with the following attributes:

1. Order ID
2. Date
3. Product Category
4. Region
5. Sales Representative
6. Revenue
7. Quantity Sold
8. Discount %
9. Order Status

The data originally existed across **multiple Excel sheets exported from POS and CRM systems**.


**Data Collection**

Sales data was collected from:

1. POS system export reports
2. CRM-based Excel reports
3. Multiple department spreadsheets

These files were consolidated into a **centralized dataset for analysis**.


**Data Cleaning & Preparation**

Data cleaning and transformation were performed using **Excel and Power Query**.

Steps included:

1. Removing duplicate records
2. Eliminating blank or incomplete rows
3. Standardizing product and category naming
4. Handling missing values
5. Validating totals against raw reports
6. Preparing the cleaned dataset for Power BI import

Power Query was used to transform and structure the data before building the analytical model.
